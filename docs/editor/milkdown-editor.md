# Editor

## 1. Overview

The editor is a WYSIWYG rich text editor hosted inside a `WKWebView`. The web layer is built on Milkdown, which is itself a wrapper around ProseMirror. Milkdown treats Markdown as its canonical document format: it parses incoming Markdown strings into a ProseMirror document tree on load, and serializes the tree back to Markdown on save. Swift reads and writes `.md` files and passes raw Markdown strings across the bridge; it never sees or interprets the internal ProseMirror representation.

The toolbar provides bold, italic, blockquote, heading levels (H1–H3), bullet list, ordered list, hyperlinks, footnote references, and image insertion. There is no underline formatting: it has no standard Markdown encoding and was dropped during the Milkdown migration.

The editor autosaves 5 seconds after a change, and also saves on ⌘S and when the editor closes. A small "Saving…" / "Saved!" island confirms save state.

Focus mode (⌘M or View → Focus Mode) hides the sidebar, constrains the toolbar and editor to a centered 820 px column, and flanks it with black gutters. ESC exits focus mode when active and "Enable in full screen by default" is OFF; otherwise it saves and closes the editor. When both focus mode and fullscreen are on, the fullscreen customizations from "Customize focus mode" activate.

## 2. Editor Source

### 2.1. File Structure

`Resources/editor.html` is generated by a Vite build — do not edit it directly. The source lives in `editor-src/`:

- `editor.html` — HTML entry point: `#focus-bg` div, toolbar markup, `#editor` div, `#footnote-tooltip` div.
- `src/main.js` — all editor logic. This is where the Milkdown editor is created and where all plugins, bridge methods, and event handlers live.
- `src/style.css` — all editor CSS: toolbar, ProseMirror content styles, footnotes, custom cursor, focus mode.
- `vite.config.js` — uses `vite-plugin-singlefile` to inline everything into a single `editor.html`. Output goes to `../BlobTxt/Resources/`.

To rebuild after editing source files:

```bash
cd editor-src
npm run build
```

### 2.2. Milkdown Packages

- `@milkdown/core` — editor factory and context system.
- `@milkdown/preset-commonmark` — CommonMark schema, commands, and keybindings. Exposes the ProseMirror mark names `strong` (bold), `emphasis` (italic), and `link`, and node names `heading`, `bullet_list`, `ordered_list`, and `blockquote`.
- `@milkdown/plugin-listener` — document and selection change callbacks with built-in debounce.
- `@milkdown/plugin-history` — undo/redo stack.
- `@milkdown/prose` — re-exports of the ProseMirror `state`, `view`, `commands`, and `model` packages.
- `@milkdown/utils` — `callCommand`, `getMarkdown`, `$node`, `$prose` utilities.
- `remark-gfm` — GitHub Flavored Markdown extension for the remark parser, needed for footnote syntax (`[^1]` and `[^1]: …`).

### 2.3. Editor Creation

The Milkdown editor is created in an async IIFE at module load:

```javascript
editor = await Editor.make()
  .config(ctx => { … })
  .use(commonmark)
  .use(listener)
  .use(history)
  .use(searchHighlightPlugin)
  .use(userInteractPlugin)
  .use(footnotePastePlugin)
  .use(footnoteReferencePlugin)
  .use(footnoteDefinitionPlugin)
  .create()
```

The `.config()` block sets the `rootCtx` to the `#editor` div, adds `remarkGfm` to `remarkPluginsCtx`, and registers the listener callbacks. After `create()` resolves, `editorReady` is posted to Swift, and the custom cursor div is appended inside `#editor`.

The listener callbacks:

- `listenerCtx.updated()` — fires with an internal ~200 ms debounce when the document content changes. Posts `documentChanged` to Swift, calls `sendStateUpdate()`, schedules `doCenteredScroll` via `requestAnimationFrame`, and calls `updateCursor()`.
- `listenerCtx.selectionUpdated()` — fires immediately on any selection move. Calls `sendStateUpdate()` and `updateCursor()`.

### 2.4. Content Get and Set

`window.editorBridge.setContent(markdown)` replaces the entire document using a ProseMirror transaction. The transaction carries `addToHistory: false`, which prevents the `updated` listener from firing and marking the document dirty. It also resets the `userInteractKey` plugin state to false. If the incoming Markdown is empty, the editor focuses and immediately sets `userInteractKey` to true so typing starts without requiring a click.

`window.editorBridge.getContent()` calls `editor.action(getMarkdown())` and then passes the result through `renumberMarkdown()` before returning the string.

### 2.5. Toolbar

The toolbar markup is in `editor.html`. The left group (`#toolbar-fmt`) contains the formatting controls; the right group (`#toolbar-chrome`) contains Copy and Close buttons.

`toolbarInitJS` (in `WebEditorView.swift`) is injected via `WKUserScript` at document-end. It wires all toolbar click handlers and drives active-state display. Because `toolbarInitJS` is a non-module script, it cannot import from `main.js` directly; instead it reads state through three globals that `main.js` exposes on `window`:

- `window.__ft_getActiveState()` — returns an object with `bold`, `italic`, `blockquote`, `link`, `heading`, `bulletList`, and `orderedList` fields, all gated on whether the user has interacted. Returns `null` before the editor is ready.
- `window.__ft_userInteracted()` — returns `true` once the user has clicked to place the cursor.
- `window.__ft_getLinkHref()` — returns the `href` of the link mark at the cursor, or `null` if none.

`toolbarInitJS` polls `__ft_getActiveState()` on a 100 ms interval to update button active classes and the heading label. Heading and list controls use dropdowns; clicks on the dropdown trigger buttons open them while closing the other; clicks anywhere on the document close both.

### 2.6. Formatting Commands

All formatting bridge methods call `editor.action(callCommand(…))` followed by `view.focus()` and `sendStateUpdate()`.

| `window.editorBridge` method | Milkdown command |
| --- | --- |
| `toggleBold()` | `toggleStrongCommand` |
| `toggleItalic()` | `toggleEmphasisCommand` |
| `toggleBlockquote()` | `wrapInBlockquoteCommand` |
| `toggleBulletList()` | `wrapInBulletListCommand` |
| `toggleOrderedList()` | `wrapInOrderedListCommand` |
| `setHeading(level)` | `wrapInHeadingCommand` (level 0 converts to paragraph) |
| `setLink(url)` | `updateLinkCommand` |
| `insertImage(src)` | `insertImageCommand` |

`unsetLink()` uses ProseMirror's `toggleMark` directly, because Milkdown's preset-commonmark does not expose a remove-link command.

Note: `wrapInBlockquoteCommand`, `wrapInBulletListCommand`, and `wrapInOrderedListCommand` wrap the current block but do not unwrap it if already active. Clicking these buttons again when the cursor is inside a list or blockquote has no effect.

### 2.7. State Reading

`sendStateUpdate()` reads the ProseMirror state and posts a `stateUpdate` message to Swift with the following fields: `bold`, `italic`, `heading` (0 or 1–3), `bulletList`, `orderedList`, `blockquote`, `linkActive`.

Two helpers query mark and block type state from a raw ProseMirror state object:

- `isMarkActive(state, markName)` — for a collapsed cursor, checks `storedMarks` then `$from.marks()`; for a range, checks `rangeHasMark`.
- `isBlockTypeActive(state, typeName, attrKey?, attrVal?)` — walks `nodesBetween` and returns `true` if any enclosing node matches the type and optional attribute value.

`getActiveHeadingLevel(state)` walks `nodesBetween` looking for a `heading` node and returns its `level` attribute, or 0.

### 2.8. Footnotes

#### 2.8.1. Schema Plugins

Milkdown has no first-party footnote support. Two `$node()` plugins define the ProseMirror schema for footnotes and bridge between the remark-gfm AST and ProseMirror:

- `footnoteReferencePlugin` — inline atom node (`group: 'inline'`, `inline: true`, `atom: true`). Carries a `label` attribute (the footnote number as a string). Renders to `<sup class="footnote-ref" data-footnote="N">[^N]</sup>`. Parses from `footnoteReference` remark AST nodes and serializes back.
- `footnoteDefinitionPlugin` — block node (`group: 'block'`, `content: 'block+'`). Carries a `label` attribute. Renders to `<div class="footnote-def" data-footnote-id="N">`. Parses from `footnoteDefinition` remark AST nodes and serializes back.

#### 2.8.2. Reference Insertion

`window.editorBridge.addFootnoteReference()` finds the highest existing numeric label in the document, assigns the next integer as the new label, and builds one `footnoteReference` and one `footnoteDefinition` node in a single transaction. The reference is inserted at the cursor position; the definition is appended at the end of the document. After dispatch, the cursor is placed inside the new definition's empty paragraph (at `defStart + 2`) so the user can type immediately.

#### 2.8.3. Renumbering

`renumberMarkdown(markdown)` is called from `getContent()`. It rewrites all footnote reference IDs to clean sequential integers in document order. The function does a first pass to discover labels in order of first appearance, then a second pass to rewrite both inline references (`[^N]`) and definitions (`[^N]:`) using a regex replacement.

This is implemented as a string operation on Markdown rather than as a remark plugin, because Milkdown's remark pipeline runs only during parsing, not during serialization.

#### 2.8.4. Paste Normalization

`footnotePastePlugin` is a ProseMirror plugin with a `transformPastedText` handler. When pasted text contains `[^` markers, it scans the current document for the highest existing footnote label and renumbers any conflicting IDs in the pasted text before Milkdown parses it. If the current document has no footnotes, the text is passed through unchanged.

#### 2.8.5. Copy Handler

A `copy` event listener on `view.dom` intercepts copies that include footnote references. It serializes the selected content as Markdown, finds all referenced definition texts, and appends them to the clipboard as `[^N]: text` lines. This prevents dangling `[^N]` markers when pasting into another document. The handler only fires when the selection contains at least one `footnoteReference` node.

#### 2.8.6. Tooltip

A `mouseover` listener on `view.dom` shows `#footnote-tooltip` when the cursor hovers over a `sup.footnote-ref` element. The tooltip's text content is set to the definition's `textContent`, which strips any inline formatting present in the definition.

A `click` listener on `view.dom` handles clicks on `sup.footnote-ref` elements by smooth-scrolling to the matching `[data-footnote-id]` div.

### 2.9. Search and Highlight

`searchHighlightPlugin` is a `$prose()` ProseMirror plugin that maintains a `DecorationSet` of `Decoration.inline` spans (class `search-highlight`). It is controlled via transaction meta:

- Setting meta to a `[{from, to}]` array replaces the decoration set with new spans.
- Setting meta to `'clear'` empties the set.
- On any other transaction the set is mapped forward.

`window.editorBridge.searchAndHighlight(query)` walks all text nodes in the document, collects `{from, to}` ranges for each match, and dispatches the meta transaction. `scrollToSearchResult(index)` uses `coordsAtPos` to find the match position and scrolls to one-third from the top. `clearSearchHighlights()` dispatches `'clear'` meta.

### 2.10. Custom Cursor

A `div#custom-cursor` element replaces the native browser text caret. `updateCursor()` is called on every selection change, focus/blur event, and click. It hides the cursor if the editor is not focused, if the selection is not collapsed, or if `userInteractKey` is false. Otherwise it uses `coordsAtPos` and `getBoundingClientRect` to position the cursor div absolutely within `#editor`. The cursor height is set to `fontSize * 1.5` px. The `blinking` CSS class is removed and re-added after a forced reflow to restart the blink animation on each cursor move.

### 2.11. User Interaction Tracking

`userInteractPlugin` is a ProseMirror plugin that tracks whether the user has clicked to establish a cursor position. Its state is a boolean; it starts as `false` after each `setContent` call. It becomes `true` on the first pointer transaction or explicit click.

The plugin's `handleKeyDown` prop returns `true` (blocking the keystroke) while the state is `false`, so typing before clicking has no effect. `window.__ft_userInteracted()` exposes this state to `toolbarInitJS` for gating the link button.

### 2.12. Centered Autoscroll

`doCenteredScroll()` runs after every `updated` callback via `requestAnimationFrame`. It does nothing unless `autoScrollMode` is `'centered'`. When active and the cursor is below the vertical midpoint of `#editor`, it smoothly scrolls the container to bring the cursor to center.

`window.editorBridge.setAutoScrollMode(m)` sets the module-level `autoScrollMode` variable and also sets `#editor`'s `paddingBottom` to `50vh` when centered mode is on (so the cursor can always reach center, even at the end of a short document).

### 2.13. Scroll Position Persistence

A debounced `scroll` event listener on `#editor` posts `{ type: 'scrollPositionChanged', scrollTop }` to Swift 300 ms after the user stops scrolling. Swift stores this value in `store.blobScrollPositions[url]` and restores it the next time the same blob is opened.

### 2.14. Focus Mode

`window.editorBridge.setFocusMode(enabled)` toggles the `focus-mode` class on `document.body`. All layout and color changes for focus mode are handled purely in CSS.

`window.editorBridge.setFocusModeCustomizations(enabled, floating, dimness, blur)` toggles `focus-custom` and `floating` classes on `document.body` and sets `--focus-dimness` and `--focus-blur` CSS variables.

`window.editorBridge.setFocusWallpaper(dataURL)` sets the `--focus-wallpaper` CSS variable to `url(…)` or `none`.

### 2.15. Globals Exposed to Toolbar Script

Because `toolbarInitJS` is injected as a non-module script, it cannot share imports with `main.js`. Three `window` properties are the interface:

- `window.__ft_getActiveState()` — returns the active formatting state, or `null` before the editor is ready. All fields except `interacted` are `false` when the user has not yet clicked.
- `window.__ft_userInteracted()` — returns the `userInteractKey` boolean state.
- `window.__ft_getLinkHref()` — returns the `href` of a link mark at the cursor, or `null`.

## 3. Swift Layer

### 3.1. EditorBridge (`Sources/Views/Editor/EditorBridge.swift`)

`EditorBridge` is an `ObservableObject` and `WKScriptMessageHandler`. It owns a weak reference to the `WKWebView` and is the single point of contact between Swift and the web editor.

#### 3.1.1. Published State

- `isReady: Bool` — becomes `true` when the `editorReady` message arrives.
- `isDirty: Bool` — becomes `true` when `documentChanged` arrives; cleared by `markClean()`.
- `editorState: EditorState` — updated on every `stateUpdate` message. `EditorState` fields: `bold`, `italic`, `heading` (Int, 0 = paragraph), `bulletList`, `orderedList`, `blockquote`, `linkActive`.
- `showLinkDialog: Bool` and `pendingLinkHref: String?` — drive the link dialog sheet.

`lastScrollPosition: Int` is not published; it is updated on `scrollPositionChanged` and read on `onDisappear` without triggering redraws.

#### 3.1.2. JS → Swift Messages

| Message | Behavior |
| --- | --- |
| `editorReady` | Sets `isReady = true` |
| `documentChanged` | Sets `isDirty = true` |
| `stateUpdate` | Updates `editorState` |
| `copyAll` | Writes Markdown plain text to the pasteboard |
| `closeEditor` | Calls `onClose?()` |
| `scrollPositionChanged` | Updates `lastScrollPosition` |
| `headingVisible` | Re-posts as `activeHeadingChanged` notification for `BlobOutlineView` |
| `insertLink` | Sets `pendingLinkHref` and `showLinkDialog = true`; payload includes `href` or `null` |
| `openURL` | Opens URL via `NSWorkspace.shared.open` |
| `insertImage` | Opens `NSOpenPanel`, base64-encodes the chosen file, calls `insertImage(src:)` |

#### 3.1.3. Swift → JS Commands

All formatting commands use `evaluate(_:)`, which calls `evaluateJavaScript` on the main thread. `insertImage(src:)` uses `callAsyncJavaScript` because the base64 payload can be large; do not swap these. Similarly, `applyFocusModeCustomizations` uses `callAsyncJavaScript` when a wallpaper URL is present so it can gate the fade-in callback on the wallpaper actually being painted.

Content passing uses `JSONSerialization.data(withJSONObject:options:.fragmentsAllowed)` to produce a properly escaped JS string literal from a Swift `String`. This handles backslashes, quotes, newlines, and non-ASCII characters without manual escaping. The result is embedded directly into the JS call as the argument.

`setContent`, `setContentAndScrollToTop`, and `setContentAndScrollTo` all follow this pattern. `getContent(completion:)` evaluates `window.editorBridge.getContent()` and returns the result as a `String?`.

Font and size changes inject a `<style id="ft-font">` element. Image half-width inject a `<style id="ft-img-style">` element. Both are idempotent: the element is created if absent, then its `textContent` is replaced.

#### 3.1.4. Notification Names

Defined as `Notification.Name` extensions in `EditorBridge.swift`:

| Name | Direction | Purpose |
| --- | --- | --- |
| `scrollToOutlineHeading` | `BlobOutlineView` → `EditView` | Scroll editor to heading at index |
| `activeHeadingChanged` | `EditorBridge` → `BlobOutlineView` | Highlight the active outline row |
| `searchAndHighlight` | `BlobSearchView` → `EditView` | Trigger in-editor search |
| `scrollToSearchResult` | `BlobSearchView` → `EditView` | Scroll to search result at index |
| `clearSearchHighlights` | `BlobSearchView` → `EditView` | Clear all search highlights |
| `reloadEditorContent` | `BlobSearchView` → `EditView` | Replace editor content from disk |
| `toggleFocusMode` | `BlobTxtApp` → `ContentView` | Toggle focus mode |
| `focusCustomizationChanged` | `FocusCustomizationView` → `EditView` | Re-apply focus customizations live |

### 3.2. EditView (`Sources/Views/Editor/EditView.swift`)

`EditView` hosts the `WebEditorView` and owns the save lifecycle. It receives `@Binding var isFocusMode: Bool` and `let isFullScreen: Bool` from `ContentView`, and a `let url: URL` identifying the blob being edited.

The root view is a `ZStack` with a surface background behind the `WebEditorView` and the save island overlaid at the bottom-trailing corner. The web view fades in after content is loaded (`contentOpacity` transitions from 0 to 1 over 0.3 s) to hide the layout settling during initial load and focus-mode transitions.

Load lifecycle: when `bridge.isReady` becomes `true`, `EditView` loads the blob content from `store`, waits 200 ms, then calls either `bridge.setContentAndScrollTo` (if a saved scroll position exists) or `bridge.setContentAndScrollToTop`. After that it calls `bridge.markClean()`, `bridge.setFocusMode()`, and `bridge.applyFocusModeCustomizations()`. The `applyFocusModeCustomizations` completion handler fires the fade-in once the wallpaper is visible (or immediately if there is no wallpaper).

Save lifecycle:

- `performSave` guards on `bridge.isDirty`, shows "Saving…", calls `bridge.getContent`, passes the Markdown to `store.saveBlobContent`, shows "Saved!" for 1.5 s, then fades to idle.
- `bridge.$isDirty` is debounced 5 seconds on `RunLoop.main`; when it fires, `performSave` is called.
- `.saveDocument` notification (⌘S or app quit) also calls `performSave`.
- `saveAndClose()` calls `performSave` and then `onClose()` in the completion.

On `onDisappear`, `store.blobScrollPositions[url]` is updated with `bridge.lastScrollPosition`.

Keyboard shortcuts registered via `NSEvent.addLocalMonitorForEvents`:

- ESC: exits focus mode if `isFocusMode && !defaultFocusMode`; otherwise calls `saveAndClose()`. Skipped when a sheet is in front.
- ⌘M: toggles `isFocusMode`.
- ⌘A: calls `bridge.selectAll()`.

Settings change handlers: `onChange(of: appColors.surface)` calls `bridge.applyColors()`; `onChange(of: fontSize)` and `onChange(of: fontFamily)` call the corresponding bridge setters.

### 3.3. WebEditorView (`Sources/Views/Editor/WebEditorView.swift`)

`NSViewRepresentable` wrapping a `WKWebView` that loads `Resources/editor.html`.

Script injection timing:

- At document-start: `AppColors.shared.editorCSSVariablesJS()` injects active palette colors as CSS variables before any CSS is computed, preventing a flash of hardcoded colors.
- At document-end: `toolbarInitJS` wires all toolbar click handlers and starts the active-state polling interval. Injecting at document-end guarantees that `window.editorBridge` and `window.__ft_getActiveState` are defined.

A `WeakMessageHandler` wraps `EditorBridge` before being added to the `WKUserContentController`. This breaks the retain cycle that would otherwise form between the controller and the bridge.

`WallpaperSchemeHandler` is registered for the `blobtxt://` URL scheme. When WebKit requests `blobtxt://wallpaper?t=<timestamp>`, the handler reads the file at `focusWallpaperPath` from `UserDefaults` on a background thread and streams the bytes back. The timestamp query parameter busts WebKit's cache when the wallpaper file is replaced.

The `Coordinator` propagates settings changes. `updateNSView` is called by SwiftUI on every view update, so each setting method guards with a `last*` variable to avoid redundant JS calls. `webView(_:didFinish:)` applies all initial settings 150 ms after the page load, because `updateNSView` is not called during the initial load.

`toolbarInitJS` also handles the outline feature: a throttled `scroll` listener on `#editor` calls `detectActiveHeading()`, which finds the heading closest to 30% from the top and posts `headingVisible`. `window.scrollToHeading(index)` scrolls the matching heading into view and fires `detectActiveHeading` on `scrollend` to update the active heading immediately.

### 3.4. LinkDialogView (`Sources/Views/Editor/LinkDialogView.swift`)

Sheet for adding or editing a hyperlink. Presented by `EditView` when `bridge.showLinkDialog` is true.

The dialog is in "edit" mode when `bridge.pendingLinkHref` is non-nil (the cursor was on an existing link). In edit mode the title reads "EDIT HYPERLINK" and a "Remove" button appears. In add mode the title reads "ADD HYPERLINK" and there is no remove button.

The URL input uses `LinkPlainTextField`, a local `NSViewRepresentable` wrapping an `NSTextField` with no bezel and no background. Pressing Return confirms the dialog.

On confirm, `https://` is prepended if the URL has no recognized prefix (`http://`, `https://`, or `mailto:`). `bridge.setLink(url:)` is called and the sheet is dismissed.

## 4. Known Limitations

The following behaviors were intentionally left as-is and are candidates for a future polish pass.

`wrapInBulletListCommand`, `wrapInOrderedListCommand`, and `wrapInBlockquoteCommand` from Milkdown's preset-commonmark only wrap; they do not toggle off. Clicking the list or blockquote button when the cursor is already inside one of those structures has no effect.

The footnote hover tooltip shows `node.textContent`, which strips any inline formatting (bold, italic) present in the definition body.

The `listenerCtx.updated()` callback has an internal ~200 ms debounce before `documentChanged` reaches Swift. The autosave timer is 5 seconds, so this delay is imperceptible in practice; it is noted in case tighter dirty-detection is ever needed.
